# Линейная алгебра

`optlib` использует собственные плотные контейнеры `Vector` и `Matrix`, потому
что по условиям проекта нельзя подключать Eigen, BLAS или Boost.

## `Vector`

`Vector` владеет непрерывным массивом `double` и предоставляет:

- размер через `Size()`;
- доступ к данным через `Data()`;
- `Span()` для передачи в алгоритмы без копирования;
- `Fill`, индексирование и конструктор из `std::span<const double>`.

Такая модель удобна для оптимизаторов: начальная точка копируется один раз,
а временные векторы переиспользуются на каждой итерации.

## `Matrix`

`Matrix` хранится в row-major порядке. Это совпадает с C-contiguous NumPy
массивами и упрощает pybind11-границу.

Основные операции:

- `At(row, col)` — индексированный доступ;
- `Rows()` и `Cols()` — размеры;
- `Span()` и `Data()` — непрерывное представление.

## Горячие операции

В ядре реализованы:

- `Dot(x, y)`;
- `Norm2(x)` и `NormInf(x)`;
- `Copy`, `Axpy`, `Scale`;
- `Gemv(A, x)`;
- `Gemm(A, B)`.

Операции проверяют размерности и выбрасывают `std::invalid_argument` при
несовпадении. Внутренние циклы простые и предсказуемые для компилятора MSVC:
без виртуальных вызовов, выделений памяти внутри арифметического цикла и
внешних библиотек.

## Связь с Python

На входе биндинги принимают C-contiguous NumPy массивы. На выходе C++ вектор или
матрица передается как NumPy view на владение через capsule, поэтому результат
можно сразу использовать в графиках и тестах.
