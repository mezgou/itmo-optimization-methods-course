# Автоматическое дифференцирование

`Dual` реализует forward-mode automatic differentiation. Одно dual-число хранит
значение и вектор частных производных по всем координатам.

Для переменной $x_i$ создается объект:

$$
v = x_i,\quad g_i = 1,\quad g_j = 0\ (j \ne i)
$$

Здесь $v$ - значение, $g$ - вектор производных.

## Распространение производных

Операции перегружены так, чтобы вместе со значением обновлять градиент.

Сложение:

$$
(u + v)' = u' + v'
$$

Произведение:

$$
(u v)' = u' v + u v'
$$

Частное:

$$
(u / v)' =
\frac{u' v - u v'}{v^2}
$$

Для элементарной функции используется цепное правило:

$$
\phi(u)' = \phi'(u) u'
$$

В ядре поддержаны `sin`, `cos`, `exp`, `log`, `sqrt`, `pow` и базовые
арифметические операции.

## Зачем это нужно

Функции benchmark пишутся шаблонно: одна реализация вычисляет значение и для
`double`, и для `Dual`. Поэтому градиент получается за один проход по выражению
без подбора шага $h$.

В Python доступны wrappers:

```python
g1 = optlib.RosenbrockAutogradGradient(x)
g2 = optlib.RastriginAutogradGradient(x)
g3 = optlib.HimmelblauAutogradGradient(x)
g4 = optlib.AckleyAutogradGradient(x)
```

## Сравнение с конечными разностями

Autograd не имеет ошибки усечения, но хранит внутри каждого числа вектор длины
$n$. Поэтому для очень больших $n$ растут память и константа времени.

Конечные разности проще применить к произвольному Python callback, но требуют
несколько вызовов функции на координату и зависят от выбора $h$.
