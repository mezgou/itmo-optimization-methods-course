# Численное дифференцирование

Модуль `Differentiation` вычисляет градиент функции многих переменных через
последовательное возмущение каждой координаты. Пользователь выбирает схему и
может явно задать шаг `h`; если шаг равен нулю, библиотека берет адаптивное
значение по масштабу координаты.

## Схемы

Для координаты `x_i` используются три формулы.

Односторонняя схема:

```text
df/dx_i ≈ (f(x + h e_i) - f(x)) / h
```

Порядок точности: `O(h)`, но нужен только один дополнительный вызов функции на
координату.

Центральная схема:

```text
df/dx_i ≈ (f(x + h e_i) - f(x - h e_i)) / (2h)
```

Порядок точности: `O(h^2)`, обычно это базовый вариант для гладких функций.

Пятиточечная схема:

```text
df/dx_i ≈ (-f(x + 2h e_i) + 8f(x + h e_i)
           - 8f(x - h e_i) + f(x - 2h e_i)) / (12h)
```

Порядок точности: `O(h^4)`, но на каждую координату требуется четыре вызова
функции.

## Выбор шага

Автоматический шаг строится как `base * max(1, |x_i|)`, где `base` зависит от
схемы:

- forward: примерно `sqrt(eps)`;
- central: примерно `cbrt(eps)`;
- five-point: примерно `eps^(1/5)`.

Это балансирует ошибку усечения и ошибку округления для `double`. В ноутбуке
первой лабораторной есть график зависимости ошибки от `h`: слишком большой шаг
портит аппроксимацию, слишком маленький усиливает округление.

## Ограничения

Численное дифференцирование предполагает детерминированную функцию. Для
разрывных black-box поверхностей градиент может быть неинформативен; в таких
случаях в следующих лабораторных используются методы нулевого порядка.
