# Линейная алгебра

Линейная алгебра реализована в `src/optlib/core/LinAlg.hpp`. Это небольшой
внутренний слой для оптимизаторов и MLP, а не замена NumPy.

## Контейнеры

`Vector` хранит одномерный contiguous буфер `double`. `Matrix` хранит плотную
матрицу в row-major порядке. Оба типа владеют памятью и дают доступ через
`std::span`, поэтому функции ядра могут работать без лишних аллокаций.

Python API не раскрывает эти контейнеры напрямую. На границе используются NumPy
массивы.

## Операции

Скалярное произведение:

$$
x^\top y = \sum_{i=1}^{n} x_i y_i
$$

Евклидова норма:

$$
\|x\|_2 = \sqrt{\sum_{i=1}^{n} x_i^2}
$$

Бесконечная норма:

$$
\|x\|_\infty = \max_i |x_i|
$$

AXPY:

$$
y + \alpha x
$$

Матрично-векторное произведение:

$$
(A x)_i = \sum_{j=1}^{m} A_{ij} x_j
$$

Матричное произведение:

$$
(A B)_{ij} = \sum_{k=1}^{m} A_{ik} B_{kj}
$$

## Python API

```python
import numpy as np
import optlib

x = np.array([1.0, 2.0, 3.0])
y = np.array([4.0, 5.0, 6.0])
A = np.eye(3)

dot = optlib.Dot(x, y)
norm = optlib.Norm2(x)
z = optlib.Axpy(0.5, x, y)
v = optlib.Gemv(A, x)
M = optlib.Gemm(A, A)
```

## Производительность

Операции написаны простыми циклами над contiguous памятью. Для учебного проекта
это важнее зависимости от внешнего BLAS: поведение прозрачно, код переносим, а
размеры задач в экспериментах умеренные.
