# Автоматическое дифференцирование

`Dual` реализует forward-mode autodiff с несколькими производными сразу:
число хранит значение и вектор частных производных по всем координатам.

## Представление

Для переменной `x_i` создается dual-число:

```text
value = x_i
gradient[j] = 1, если i == j
gradient[j] = 0, если i != j
```

Дальше перегруженные операции распространяют производные по правилам
дифференцирования. Например, для произведения:

```text
(u * v)' = u' * v + u * v'
```

Для функций `sin`, `cos`, `exp`, `log`, `sqrt`, `pow` применяются обычные
цепные правила.

## Зачем это нужно

Для функции Розенброка шаблонная реализация `Eval<T>` работает и с `double`, и
с `Dual`. Поэтому градиент можно получить за один проход по выражению без
подбора шага `h`.

Плюсы:

- нет ошибки конечных разностей;
- удобно проверять аналитический градиент;
- один API для 2D и `N`-мерных задач.

Минусы:

- forward-mode хранит вектор производных внутри каждого числа;
- для очень большой размерности память и константа времени растут быстрее, чем
  у ручного аналитического градиента.

В первой лабораторной autograd используется как точный эталон рядом с
аналитическим градиентом и численными схемами.
